Memory Navis

Group Project together with 5 other students (4 artists and 2 programmers, myself included),
this project was build for a client and mainly focuses on the integration with Eno Phones or other EEG scanners.

Project Size

12 work weeks (8:30 to 17:00), 5 team members

Tools & Languages

Unity, C#, Perforce, Hack'n'Plan

Link Project

Itch.io

Discription

Neural connections have become more and more common in this day and age, people have been uploading themselves into machines as Human Intelligence. Some chose to do it while others were not given the choice.
You are monitoring ships for the Argus company, trying to fix the uncontrolled H.I ships. You use the Eno Phone technology to get a closer connection to this intelligence and guide them.
While looking into a ship that deviaded from it's path you discorver new things about the vessel and why it got uploaded in there. Will you help this ship become it's old self again or will you send it back to Argus for repairs, thus deleting the original Human Intelligence.

Originally, the game is meant to be played with the Eno Phones or other EEG scanners. If you do not own such device the game will be played with random data instead of your brain activity.

My Principal Contributions

Trail Following Steering Behavior

I made the trail steering behavior so that you can let the Spark (the flooding energy ball) follow a certain path. The Spark remains just out of range so that the player thinks he can still catch it. However, only when the Spark finishes the path it becomes possible to catch.

Dialoge System

I added dialoge functionality to an existing dialoge pack we used. Adding dialoge triggers, dialoge speed, dialoge sounds, etc...

Performance

Because we were using volumetric fog and a lot of debris in the performance sometimes dropped to an unacceptable levels. I managed the performance (optimizing the volumetric fog, creating an object pool for debris, discussing texture sizes with the artist, etc...).
